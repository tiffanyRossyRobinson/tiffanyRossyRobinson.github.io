Just For The Halibut!

Background:

This app served as my final project for the Font-End Engineering course at The Iron Yard. It was a group project that consisted of a classmate (Trevor Glass) and myself.

What Does It Do?

Both my team mate and I are from areas that are not surrounded by water and don't host alot of fishing. We both recently found ourselves in Charleston where there is alot of fishing! This app allows users to post the fish they have caught in their area, thus allowing other users to view what people are catching in the locations they are not familiar with. In addition, the app has been gamified to allow fishers to gain points for the catches they post, using an algorithm we generated, that takes into account the size and species of the fish. Fishers are able to see the top three fishers in the app and their top catch.

Personal Reflection:

I had a blast working on this project! Learing how to use Transloadit to allow users to upload photos was important for the app and my experience. I felt like we were able to accomplish so much with this app and even now I still have ideas on new features to add. Looking back, I would spend more time sitting and brainstorming these features to better organize how the data is stored.

Technologies:

Angular, Node, Transloadit, Angular Bootstrap, Angular Google-Maps, ngGeolocation, Font-Awesome, Underscore, Moment, Food2Fork API, Javascript, SASS, CSS, HTML

PlaceMe

Background:

This app was the second group project I particapated in the Front-End Engineering program with the Iron Yard. The group memebers were classmates Mark Kimball, Mitchell Hunt, and myself.

What Does It Do?

Anyone who loves to travel knows how long it can take to search the web and find locations they want to visit/move to. With this app, you can cut down the time spent researching destinations. Using an algorithm we created, this app runs the user through a series of questions then processes their response to suggest 3 cities they would fit best. Based on their interest, this fun quiz allows the user to discover cities they never would have thought of!

Personal Reflection:

I was very pleased with the way this app came out. There was alot of javascript involved, but I found it fun and exciting! The restriction in this project was having to generate our own database of cities. Having learned about APIs since this project, I would love to go back and use an API (or multiple if needed) to allow a wider database to place users.

Technologies:

Jquery, Javascript, Bootstrap, Underscore, Font-Awesome, CSS, HTML

Powell Peralta

Background:

The assignment to create this web page was given in the first week of our course. The project introduced us to the concept of making responsive web pages that could accomodate multiple devices.

What Does It Do?

This app is a Powell Peralta inspired web page using HTML and CSS. It focuses on the design of web pages that is responsive and adjust based on the screen size of the users device.

Personal Reflection:

Although the web page primarily uses HTML, CSS, and SASS, I am very proud and happy with this project. It allowed me to take time and focus on the design and layout of the web page where other projects I worked on were heavy on Javascript or other more complex technologies. I look back on this piece to remind myself of where I was when I started and to see how much I have grown as a developer.
